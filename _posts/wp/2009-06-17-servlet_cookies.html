一、创建cookies

　　用cookies的构造函数，cookies(String name,String value)，需要注意的是参数名不能含有空格及下列字符：[ ] ,( ) = ” / ? @ : ;

二、 读取cookies和设定cookies属性

　　在将cookies加入到输出标题之前，可以取得并且设定cookies的属性，具体函数如下：

函数 说明

getComment/setComment 得到/设置 cookie的注释

getDomain/setDomain 得到/设置 cookies应用的域名。通常cookies只会返回到发送该cookies的主机上。用该方法可以通知浏览器将cookies返回到指定的在同一域的其他主机上。

getMaxAge/setMaxAge 得到/设置cookie有效时间期间，默认为此次连接，及直到关闭该页面，cookies不会存放在硬盘上。

getName/setName 得到/设置cookie名。 名称与值是开发者最为关心的，用HttpServletRequest 得到Cookie 对象数组，通常就循环调用getName，直到得到想要得到的cookies，再用getValue得到其中的值

getPath/setPath 得到/设置cookies应用的目录。如果不指定与该cookie 将适用于与当前页面在同一目录以及该目录下的所有子目录的文件。

getSecure/setSecure 得到/设置 是否cookies应该按照加密方式传送

getValue/setValue 得到/设置cookie的值

getVersion/setVersion 得到/设置cookie 的版本号，默认为0。

添加和显示Cookie

本节的代码如下：

package com.kettas.servlet;

import java.io.*;

import javax.servlet.*;

import javax.servlet.http.*;

public class HelloCookie extends HttpServlet {

public void service(HttpServletRequest request,

HttpServletResponse response) throws ServletException,

IOException {

String title = "Display all cookie and create a cookie";

String heading1 = "Cookie information is:";

Cookie theCookie = null;

Cookie cookies[] = request.getCookies();

String cookieKey = request.getQueryString();

System.out.println("QueryString= " + cookieKey);

response.setContentType("text/html");

PrintWriter out = response.getWriter();

//find if queryString input.

if (cookieKey != null) {

String cookieValue = cookieKey + "Value";

theCookie = new Cookie(cookieKey, cookieValue);

theCookie.setMaxAge(10);

response.addCookie(theCookie);

}

// write data or building web page.

out.print("");

out.print("");

out.print("

" + heading1 + "

");

if (cookies != null) {

for (int i = 0, n = cookies.length; i < n; i++) {

out.println("cookie.getName()/getValue: "

+ cookies[i].getName()

+ " / " + cookies[i].getValue()

+ "
cookie.getMaxAge(): "

+ cookies[i].getMaxAge()

+ "
cookie.getDomain(): "

+ cookies[i].getDomain()

+ "
cookie.getPath(): "

+ cookies[i].getPath()

+ "
cookie.getVersion(): "

+ cookies[i].getVersion()

+ "

");

}

}

out.print("");

out.flush();

}

}

该程序首先生成Cookie，它把浏览器输入的QueryString作为Cookie的名字，然后把这个名字加上字符串Value作为Cookie的值，具体的代码和讲解如下：

//建立Cookie对象，其中cookieKey是浏览器输入的QueryString

//cookieValue在浏览器输入的QueryString后加上Value

//关于QueryString，在运行代码时就会看到

theCookie = new Cookie(cookieKey, cookieValue);

//设置Cookie的生命期为10秒

theCookie.setMaxAge(10);

//把Cookie作为response的一部分，写入到用户浏览器中

response.addCookie(theCookie);

如果要显示用户浏览器中保存的Cookie，首先用下面的代码得到Cookie数组：

Cookie cookies[] = request.getCookies();

然后利用循环，调用cookies[i].getXXX()方法，得到Cookie的信息。

运行本节代码的网址如下：

http://zhb:8080/ServeltCookie/hellocookie?ant

其中问号后的内容就是QueryString，可自己选择，如果输入ant，那么就建立一个name为ant、value为antValue的 Cookie。读者注意QueryString和request对象的parameter不同，前者指问号的所有内容，后者的格式必须是?name1=value1&name2=value2。

第一次调用上面的网址时，浏览器中可能没有Cookie，当刷新网页时，就会显示第一次运行时加入的Cookie(ant/antValue)，可能显示的内容如下：

Cookie information is:

cookie.getName()/getValue: Name / zhb

cookie.getMaxAge(): -1

cookie.getDomain(): null

cookie.getPath(): null

cookie.getVersion(): 0

————————————————————————

cookie.getName()/getValue: ant / antValue

cookie.getMaxAge(): -1

cookie.getDomain(): null

cookie.getPath(): null

cookie.getVersion(): 0

读者在自己的电脑上运行时，显示的其他Cookie可能与本书的不同。

第一次用某个QueryString如any时，用户浏览器不会立刻显示这个写入的Cookie(至于为什么，留给读者思考)，只有再次刷新后才能看到。

程序中的setMaxAge方法设置Cookie的生命期，该方法的参数是秒，读者可以把这个参数修改得比较小，例如设置生命期为10秒，10秒后刷新浏览器时就会发现Cookie不存在了。

如果设置Cookie的生命期为0，那么表示删除这个Cookie。

如果生命期为负数，表示不永久保存Cookie，当浏览器退出时就删除掉Cookie。

另外，写入的Cookie到一定数量后可能就不能再写了，这时只要删除Windows安装目录中的Temporary Internet Files子目录的内容，以后就可再次写入了。

在Windows安装目录中的Cookies子目录中，也有写入的Cookie的信息，读者可以打开里面的文件看看。
